  /admin/services     Manage service cards

How content changes propagate:
  1. Admin edits text in /admin/content
  2. Clicks "Save [Section]"
  3. POST /api/admin/content → upserts rows into site_content table
  4. Next page load of the website: layout.tsx fetches updated rows
  5. LangProvider receives overrides, t() returns new values
  6. No redeploy needed — changes are live immediately on next request

To reset a key to the static default:
  Delete the corresponding row from site_content in Supabase dashboard.

8. I18N / TRANSLATIONS SYSTEM
────────────────────────────────────────────────────────────────────────────────

Translation keys format: "section.subkey"  (e.g. "hero.title", "faq.q1")

Adding a new key:
  1. Add to lib/i18n/translations.ts: 'my.key': { PT: '...', EN: '...' }
  2. Use in component: const { t } = useLang(); ... t('my.key')
  3. It will automatically appear in /admin/content under the matching section tab

CMS override priority:
  DB (site_content) > static translations.ts > key string (fallback)

Language toggle:
  Stored in localStorage under key 'webdigio-lang' ('PT' or 'EN')
  Toggle button in Navbar top right

9. ANIMATION SYSTEM
────────────────────────────────────────────────────────────────────────────────

GSAP Rule: Always dynamically import GSAP to avoid SSR crashes:
  const { gsap } = await import('gsap');
  const { ScrollTrigger } = await import('gsap/ScrollTrigger');
  gsap.registerPlugin(ScrollTrigger);

HeroPortal scroll animation:
  - Progress 0 → 1 via window.scrollY / window.innerHeight
  - applyProgress(): scales device, reveals inner content
  - Dispatches CustomEvent('portal-complete') at progress >= 0.88
  - Dispatches CustomEvent('portal-reset') when going back below
  - Navbar listens to these events (show/hide)

CTAFooter curtain reveal (21st.dev pattern):
  - Wrapper div: h-screen, clipPath: polygon(0% 0, 100% 0%, 100% 100%, 0 100%)
  - Footer: position:fixed, bottom:0, height:100vh
  - As page scrolls to wrapper, clip reveals fixed footer naturally
  - GSAP ScrollTrigger adds parallax on giant "WEBDIGIO" text

MagneticButton:
  - useRef on anchor/button element
  - GSAP mousemove: translate + rotationX/Y toward cursor
  - GSAP mouseleave: elastic.out(1, 0.3) spring back

11. SUPABASE AUTH / MIDDLEWARE
────────────────────────────────────────────────────────────────────────────────

middleware.ts handles /admin/* routes:
  - Unauthenticated + not on login page → redirect to /admin/login
  - Authenticated + on login page → redirect to /admin
  - Uses @supabase/ssr createServerClient with request/response cookies

Session management:
  - Supabase uses HttpOnly cookies managed by @supabase/ssr
  - Server components use createSupabaseServerClient() (from lib/supabase/server)
  - Client components use createSupabaseBrowserClient() (from lib/supabase/client)
  - Admin writes use createSupabaseAdminClient() (service role, bypasses RLS)

Creating new admin users (via PowerShell):
  $h = @{ "apikey" = "<service_role_key>"; "Authorization" = "Bearer <service_role_key>"; "Content-Type" = "application/json" }
  Invoke-RestMethod -Method POST -Uri "https://qbcqsdakghzmcmqetuac.supabase.co/auth/v1/admin/users" -Headers $h -Body '{"email":"user@example.com","password":"Pass123!","email_confirm":true}'

Resetting a user's password:
  $userId = "<user-uuid>"
  Invoke-RestMethod -Method PUT -Uri "https://qbcqsdakghzmcmqetuac.supabase.co/auth/v1/admin/users/$userId" -Headers $h -Body '{"password":"NewPass123!"}'

13. KNOWN ISSUES & NOTES
────────────────────────────────────────────────────────────────────────────────

[WARN]  "self is not defined" in middleware.js — occurs during hot reload in
        dev mode when middleware is evaluated in Node context. Harmless in
        production (middleware runs in Edge Runtime where self is defined).

[WARN]  Image width/height warnings — Next.js Image components with CSS
        width overrides need width="auto" or height="auto" style prop.
        Non-breaking warning.

[INFO]  Fast Refresh performs full reload when translations.ts is modified
        (it's imported by both client and non-client modules). Normal behavior.

[INFO]  metadataBase not set — affects OG image URL resolution. Set
        metadataBase in app/layout.tsx metadata when deploying to production:
        metadataBase: new URL('https://webdigio.pt')

[INFO]  site_content RLS policy uses auth.role() = 'authenticated' for writes.
        The API route uses the service role client to bypass this, so the
        authenticated user check is done at the application level (not RLS).

[INFO]  Translation keys used in CTAFooter (footer.*) use t() from useLang().
        Some keys (cinematic.badge3/4) were removed from t() calls due to
        missing translation entries — now use direct lang === 'PT' checks.
